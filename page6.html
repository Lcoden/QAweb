מסמכי בדיקות

STP - Software Test Plan
מסמך תכנון הבדיקות :
TEST NAMEשם הבדיקה
TEST PURPOSEמטרת הבדיקה
TEST SUBJECTנושא הבדיקה
PRE CONDITIONSתנאים מוקדמים לביצוע הבדיקה
COMMENTSהערות
ATTACHMENTSקישורים למסמכים נוספים במידת הצורך
STD - Software Test Description
תרחיש בדיקה = מהלך הבדיקה :
  • STEP (צעדים) - הנחיות מדויקות לאיך לבצע את הבדיקה.
  • התוצאה הצפויה - מה אמור לקרות כשנעשה צעד מסוים.
  • תוצאה בפועל
  • האם הצעד עבר/נכשל
  • הערות

*הרצת בדיקות: ביצוע בפועל של מה שתוכנן ונכתב בשלב התכנון וכתיבת הבדיקות.

STR - Software Test Report

פורמט מובנה שמעביר בצורה מסודרת את כל המידע.

פרמטרים שיכללו בדיווח:

  • נדווח על הבדיקות הספציפיות שנעשו.
  • כמות התקלות שנמצאו, רמת החומרה, רמת הדחיפות, איפה נמצאות התקלות, השוואה בין כמות הבדיקות בסבבים, המלצות להמשך.

קריטריונים לניתוח תוצאות:

  • ביצוע בדיקות בפועל - כמה בדיקות בוצעו מתוך הבדיקות שנכתבו.
  • אחוז הבדיקות שעברו בהצלחה.
  • כיסוי דרישות - כמה דרישות כוסו.
